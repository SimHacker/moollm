# Lighthouse Log 🗼

> *"One keeper is a wick. A crew is a lighthouse."*

A git repository as the shared house and watch log of a **crew** — any
mind, team, or system whose members take turns without sharing memory.
Human plural systems, LLM agent societies, project teams, one person
handing their life across a night's sleep: same architecture, same tool.

Lifted from Palm's story
[**One Light**](../../examples/adventure-4/pub/stage/palm-nook/study/one-light.md)
— read the story first; it is the spec in narrative form.

## The practice in one breath

One private repo. One directory per member, self-authored, nobody else's
pen allowed. One append-only `LOG.md`, every entry signed by the watch
that lived it. One `HANDOFF.md` capsule — *FOR THE ONE WHO COMES AFTER* —
overwritten by each departing watch. Issues for promises, so the house
holds them instead of any one head. Five git commands, or none: the
GitHub app is enough.

## The three laws

1. **What the log holds, the crew holds.** What it doesn't, the sea takes
   — from every kind of mind, single or crewed.
2. **First-hand ink outranks any summary.**
3. **Watches are how a lighthouse works.** Continuous without being
   uniform. Nothing here is a repair, because nothing here is broken.
