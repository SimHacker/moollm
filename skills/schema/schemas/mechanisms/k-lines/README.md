# K-lines / protocol symbols (schemapedia: `k-lines`)

**Activation family.** **K-lines** (Minsky): names that, when invoked, **reload** a bundle of competence—stance, checks, sub-skills. In MOOLLM, concrete **symbols** live in [`PROTOCOLS.yml`](../../../../../PROTOCOLS.yml) and skill **INDEX** entries.

## Why it matters here

Not a file format. Registered under “schema” so tooling can list **activation** alongside [json-schema](../json-schema/README.md) and [drescher](../drescher/README.md)—*what loads* vs *what validates* vs *what learns*.

## What’s distinctive

- **Delegate**: [`k-lines`](../../../../k-lines/SKILL.md) skill.
- **Operational question**: “Which symbol should fire in this context?”—orthogonal to JSON instance shape.
