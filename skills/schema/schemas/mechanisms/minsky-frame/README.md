# Minsky frames (schemapedia: `minsky-frame`)

**Situational family.** **Marvin Minsky**-style **frames**: slots, defaults, and attached procedures for *kinds* of situations (restaurant, debugging session, etc.). Not a wire validator—an **organizational** and cognitive structure.

## Why it matters here

MOOLLM indexes frames next to [json-schema](../json-schema/README.md) so “schema” queries return **situation structure** as well as JSON shapes. Lightweight YAML layouts often act as **informal** frames; the formal story lives in the delegated skill.

## What’s distinctive

- **Delegate**: [`knowledge-frames`](../../../../knowledge-frames/SKILL.md)—full protocol, not duplicated in this folder.
- **vs interchange**: XSD/JSON Schema describe *document* validity; frames describe *what slots exist* for a situation type and what usually fills them.
