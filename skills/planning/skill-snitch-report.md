## Executive Summary

**Flexible task decomposition and progress tracking.**

Less rigid than plan-then-execute — plans can evolve.

---

## Methods

| Method | Purpose |
|--------|---------|
| **PLAN** | Create a task plan |
| **DECOMPOSE** | Break step into sub-steps |
| **PROGRESS** | Mark step as done |
| **STATUS** | Show plan progress |
| **REVISE** | Update the plan |

---

## Plan Schema

```yaml
goal: "What to accomplish"
steps:
  - "Step 1"
  - "Step 2"
progress:
  "Step 1": done
  "Step 2": in-progress
```

---

## Flexibility

> "Plans can evolve."

- DECOMPOSE when you understand more
- REVISE when circumstances change
- PROGRESS as you go
- STATUS to see where you are
